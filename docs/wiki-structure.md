# Wiki Folder Structure

Inspired by the InfraNodus/Karpathy LLM Wiki approach.
Each namespace follows the same internal structure.

```
wiki/
  @darius/
    concepts/       Core concepts — one page per idea, heavily linked
    sources/        Source summaries with backlinks to concepts
    data/           Structured evidence, numbers, references
    output/         Generated insights, research questions from gap analysis
    gaps/           Gap analysis reports (auto-generated by Kai)
    reisen/         Personal: travel pages (geo/track ingests land here)
    notizen/        Personal: free notes
    projekte/       Personal: project pages
  @soenke/
    (same structure)
  groups/
    hiking/
    projekte/

inbox/              Short-term memory (never synced, TTL 30d)
  emails/
  notes/
  claude-sessions/
  triggers/

review/             Staging — LLM proposals awaiting user approval
  candidates/
```

## Page types

### `concepts/`
One page per concept. Heavily interlinked. The knowledge graph
is built primarily from links between concept pages.

Example: `concepts/crdt.md`, `concepts/gossipsub.md`, `concepts/did.md`

Template:
```markdown
---
title: CRDT
tags: [concept, sync, distributed]
---

# CRDT (Conflict-free Replicated Data Type)

One-paragraph definition.

## How it works
...

## Related
- [[yjs]], [[automerge]], [[gossipsub]]
```

### `sources/`
Summaries of external sources (articles, papers, videos, books).
Each source links back to the concepts it supports.

Template:
```markdown
---
title: "Fix Karpathy's LLM Wiki with a Knowledge Graph"
source: https://...
ingested: 2026-04-15
tags: [source, llm-wiki, knowledge-graph]
---

# Fix Karpathy's LLM Wiki with a Knowledge Graph

**Key takeaways:**
- ...

**Links to concepts:** [[llm-wiki]], [[infranodus]], [[gap-analysis]]
```

### `output/`
Research questions and insights generated from gap analysis.
Created by Kai after running `wiki_gaps`.

Template:
```markdown
---
title: "Gap: CRDT ↔ DID identity"
generated: 2026-04-15
gap-clusters: [sync, identity]
---

# Research Question: How do CRDTs interact with DID-based identity?

Gap identified between the "sync" cluster and the "identity" cluster.

**Prompt used:** Based on the gap between "CRDT" and "DID"...

**Kai's exploration:**
...
```

### `gaps/`
Auto-generated gap reports from `wiki_gaps`. Archived for reference.

## The knowledge graph workflow

```
1. Ingest sources → sources/ + concepts/ (with wikilinks)
        │
        ▼
2. wiki_graph     → see cluster structure
        │
        ▼
3. wiki_gaps      → find unconnected clusters + research prompts
        │
        ▼
4. Kai explores   → writes to output/
        │
        ▼
5. New pages      → add concepts/, update sources/
        │
        └──► repeat (graph evolves)
```

The key insight: wikilinks (`[[page]]`) are the edges of the graph.
The more intentionally you link, the more useful the gap analysis becomes.
